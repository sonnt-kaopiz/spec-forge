# 04.2 — Command: /forge:map-codebase

**Phase**: 1 - Foundation
**Priority**: High
**Depends on**: 04.1
**Plan reference**: forge:map-codebase workflow

## Description

Create the `/forge:map-codebase` slash command. This command orchestrates four parallel `codebase-mapper` agent instances — one per focus area — that each write their own document directly to `.ai-workflow/codebase/`. The orchestrator receives only confirmation and line counts from each agent, then writes a summary index.

## Deliverables

- [ ] `commands/forge/map-codebase.md` — command prompt that:
  1. Determines `service_root` (defaults to cwd, accepts optional path argument)
  2. Ensures `.ai-workflow/codebase/` directory exists under `service_root`
  3. Spawns four `codebase-mapper` agents **in parallel**, each with:
     - `focus_area`: stack | integration | architecture | structure
     - `output_path`: `.ai-workflow/codebase/<focus_area>.md`
     - `service_root`: resolved service root path
  4. Collects results `{ status, lines, path }` from all four agents
  5. Writes `.ai-workflow/codebase/index.md` — a summary with links to each doc, line counts, and a one-line description of each focus area's scope
  6. Reports completion to the user: which files were written, total lines, any agent failures

## Output Files

```
.ai-workflow/codebase/
├── index.md         ← orchestrator writes this (summary + links)
├── stack.md         ← written by codebase-mapper (stack focus)
├── integration.md   ← written by codebase-mapper (integration focus)
├── architecture.md  ← written by codebase-mapper (architecture focus)
└── structure.md     ← written by codebase-mapper (structure focus)
```

## index.md Format

```markdown
# Codebase Map — <Service Name>
_Generated: <timestamp>_

| Document | Focus | Lines |
|----------|-------|-------|
| [Stack](stack.md) | Languages, frameworks, dependencies, config | N |
| [Integration](integration.md) | Services, databases, auth | N |
| [Architecture](architecture.md) | Patterns, layers, data flow, entry points | N |
| [Structure](structure.md) | Directory layout, key locations, conventions | N |
```

## Design Constraints

- The orchestrator command does **not** read or process the content of agent-written documents
- If an agent fails, the orchestrator logs the failure and continues — partial maps are valid output
- Command is idempotent: re-running overwrites existing docs in `.ai-workflow/codebase/`
- No state.yaml involvement — this command is stateless and can run at any time

## Acceptance Criteria

- Command runs four agents in parallel (not sequentially)
- Each agent writes its document independently
- `index.md` is generated only after all agents report back
- Command output tells the user exactly which files were created and how large they are
- Command works from any service repo root, not just spec-forge itself
