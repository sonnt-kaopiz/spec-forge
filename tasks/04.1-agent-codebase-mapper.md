# 04.1 — Agent: codebase-mapper

**Phase**: 1 - Foundation
**Priority**: High
**Depends on**: 01
**Plan reference**: forge:map-codebase workflow

## Description

Define the `codebase-mapper` subagent. This agent is instantiated in parallel — one per focus area — during `forge:map-codebase`. Each instance receives a specific focus area and a target output path, explores the codebase independently, and **writes the document file directly**. It reports back only confirmation and line count.

## Deliverables

- [ ] `agents/codebase-mapper.md` — subagent prompt with:
  - Input: `focus_area` (stack | integration | architecture | structure), `output_path`, `service_root`
  - Behavior per focus area:
    - **stack**: languages, runtimes, frameworks, dependencies (composer.json, package.json, etc.), environment config (.env.example, config files)
    - **integration**: inter-service communication (HTTP clients, queues, events), databases (connections, schemas), authentication mechanisms
    - **architecture**: design patterns, layering (MVC/DDD/Hexagonal), data flow, key abstractions, primary entry points
    - **structure**: directory layout, key locations (routes, controllers, models, tests), naming conventions
  - Output behavior: writes structured markdown to `output_path`, then returns `{ status: "done", lines: <N>, path: "<output_path>" }`
  - Self-contained: includes all exploration instructions without relying on external context

## Output Document Format

Each mapper agent writes a markdown file with these sections:

```
# <Focus Area> — <Service Name>
_Generated: <timestamp>_

## Summary
One-paragraph overview of findings.

## Details
Structured findings, using headers, lists, and code blocks as appropriate.

## Key Files
Bulleted list of the most important files for this focus area.
```

## Acceptance Criteria

- Agent prompt is self-contained (no CLAUDE.md dependency)
- Agent writes the output file directly using Write tool
- Agent returns structured confirmation (status + line count) after writing
- Prompt clearly scopes each focus area so parallel instances don't overlap
